Topbar & Sidebar

Topbar

The topbar is the fixed header you see at the top. It always contains:

Structure

  • ProDocStore logo - always present, left-aligned, links to index.html. Uses logo.svg with class .topbar-logo
  • Badge - project name in uppercase, accent-colored
  • Hamburger menu - appears on mobile (<900px)

HTML

<header class="topbar">
  <button class="menu-toggle" ...>&#9776;</button>
  <h1>
    <a href="index.html"><img src="logo.svg" alt="ProDocStore" class="topbar-logo"></a>
    <span class="badge">YOUR PROJECT</span>
  </h1>
</header>

Topbar Navigation (Multi-Page Sites)

When a site has multiple pages, add a <nav class="topbar-links"> inside the header. The core rule:

Topbar = which page, sidebar = where on this page

The topbar switches between pages. The sidebar navigates within the current page. Never mix the two. The logo/badge always links to index.html - do not add a separate "Home", "Overview", or "Knowledge Base" text link in the topbar. Clicking the logo is how you go home.

Pattern A: Flat Links (up to ~7 pages)

For sites with a manageable number of pages, use flat links directly in the topbar.

<nav class="topbar-links">
  <a href="process.html">Process</a>
  <a href="pricing.html" class="active">Pricing</a>
  <a href="team-models.html">Team Models</a>
  <a href="portfolio.html">Portfolio</a>
  <a href="glossary.html">Glossary</a>
  <a href="changelog.html">Log</a>
</nav>

Example: a knowledge base, an internal handbook

Pattern B: Dropdowns (8+ pages)

When a site grows beyond ~7 pages, group related pages into dropdown menus using nav.css. Keep top-level items to 5–7 (mix of dropdowns and flat links).

<!-- Add nav.css alongside styles.css -->
<link rel="stylesheet" href="nav.css">

<nav class="topbar-links">
  <div class="nav-drop">
    <a href="#" class="nav-drop-trigger">Product</a>
    <div class="nav-drop-menu">
      <a href="concept.html">Concept</a>
      <a href="dimensions.html" class="active">Dimensions</a>
      <a href="factors.html">Factors</a>
    </div>
  </div>
  <div class="nav-drop">
    <a href="#" class="nav-drop-trigger">Build</a>
    <div class="nav-drop-menu">
      <a href="architecture.html">Architecture</a>
      <a href="privacy.html">Privacy</a>
    </div>
  </div>
  <a href="competitors.html">Competitors</a>
  <a href="glossary.html">Glossary</a>
  <a href="changelog.html">Log</a>
</nav>

Examples: a product site (Platform, Identity, Risk dropdowns), a workflow handbook (Product, Build dropdowns)

Copy templates/nav.css into your docs/ directory and link it after styles.css. The CSS handles hover/focus reveal, the down-arrow indicator, and dark-themed dropdown menus.

Guidelines

Do

  • Keep link labels short (1–2 words)
  • Limit visible top-level items to 5–7 (use dropdowns to stay under)
  • Set class="active" on the link matching the current page (inside dropdown menus too)
  • Use the same topbar HTML on every page in the site
  • Let the logo/badge be the only way to go home

Don't

  • Add an "Overview", "Home", or "Knowledge Base" link - the logo does this
  • Put section anchors (#something) in topbar - those belong in the sidebar
  • Use dropdowns with only 1–2 items - keep them flat
  • Exceed ~4 items per dropdown - split into two groups or promote some to flat links
  • Forget to update topbar links on all pages when adding a new chapter

Single-Page vs Multi-Page

Single-Page Site (no topbar links)

One index.html. All content in sections. Sidebar handles all navigation.

Examples: a customer-facing knowledge base, an internal reference site

Multi-Page Site (with topbar links)

Multiple .html files. Topbar links switch between pages. Sidebar handles within-page navigation.

Examples: small reference site (flat), engineering handbook (flat), product docs (dropdowns), commercial site (flat)

Sidebar Navigation

The sidebar groups links under uppercase category labels. Each link targets a <section id="..."> on the current page. The active link highlights automatically as you scroll.

Index page sidebar

On the main index.html, the sidebar groups the major sections of the overview page.

<nav class="sidebar" id="sidebar">
  <div class="nav-group">
    <div class="nav-group-title">Overview</div>
    <a href="#about">About</a>
    <a href="#problem">Problem</a>
    <a href="#solution">Solution</a>
  </div>
  <div class="nav-group">
    <div class="nav-group-title">Resources</div>
    <a href="#kb-map">KB Map</a>
  </div>
</nav>

Sub-page sidebar

On sub-pages, the sidebar shows an "On this page" group with anchors for that page's sections. No back link is needed - the logo and topbar handle navigation between pages.

<nav class="sidebar" id="sidebar">
  <div class="nav-group">
    <div class="nav-group-title">On this page</div>
    <a href="#overview">Overview</a>
    <a href="#details">Details</a>
    <a href="#status">Status</a>
  </div>
</nav>
No back links in sidebar

Do not add a "← Knowledge Base" or "← Overview" link in the sidebar. The topbar and logo already provide navigation between pages. Adding back-links clutters the sidebar and creates an inconsistency - is it a section anchor or a page link? Keep it clean: sidebar = anchors on this page, only.

Cards & Grids

Cards are the primary content container. Use them to group related information.

Basic Card

A standard card with a border. This is one:

Use .card for general content blocks. Add a heading with <h4> and content below it.

Highlighted Card

Add .highlight for emphasis - accent-colored border and subtle background tint:

This Is a Highlighted Card

Use .card.highlight for important content, key takeaways, or primary actions.

Grid Layouts

Place cards in a grid using .grid-2 (two columns) or .grid-3 (three columns). Grids collapse to single column on mobile.

Grid Column 1

.grid-3 gives you three equal columns.

Grid Column 2

Each child element becomes a column.

Grid Column 3

HTML

<div class="grid-2">
  <div class="card">...</div>
  <div class="card">...</div>
</div>

<div class="grid-3">
  <div class="card">...</div>

</div>

Tables

Use tables for structured data: tech stacks, environment variables, API endpoints, comparisons. Always wrap in .table-wrap for horizontal scroll on small screens.

ClassPurposeWhen to Use
.table-wrapScrollable containerAlways - wrap every table
<th>Column headerAlways - first row of every table
<td>Data cellFor each piece of data

HTML

<div class="table-wrap">
  <table>
    <thead><tr><th>Column 1</th><th>Column 2</th></tr></thead>
    <tbody>
      <tr><td>Data</td><td>Data</td></tr>
    </tbody>
  </table>
</div>

Typography

Consistent heading hierarchy keeps content scannable.

ElementStylingUsage
<h2>22px, white, accent underlineSection titles - one per <section>
<h3>17px, accent-coloredSubsection titles within a section
<h4>15px, whiteCard headings, small groupings
<p>15px, light grayBody text
<code>13px, monospace, dark bgInline code, file names, commands
<pre><code>Block code with paddingCode examples, shell commands
Blockquotes use a left accent border and italic text. Use them for important notes or quotes.

Tags & Callouts

Tags

Small inline labels for categorization:

Primary - use .tag.tag-primary for core/important items
Secondary - use .tag.tag-secondary for supplementary items
Category - use .tag.tag-lang for categories or tech labels

Callouts

For important notes that need to stand out from normal content:

This is a callout

Use .callout with a .callout-label for key information, tips, or warnings. It has an accent border and subtle background.
