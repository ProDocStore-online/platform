What Is This

ProDocStore is an open-source documentation standard for the agent era. Your AI writes code; this is how it documents what it built so humans can read, approve, and steer.

Hand-written HTML. Zero build step. A browser extension that reads the same source agents do. This site is itself built with the template it documents - what you see here is what every ProDocStore-enabled site looks like.

What It Covers

  • Site layout and navigation structure
  • UI components (cards, tables, tags)
  • Content structure and section order
  • Voice, tone, and formatting rules

How to Start a New Site

  1. Copy docs/index.html from ProDocStore
  2. Change the badge text and page title
  3. Replace sidebar nav groups with your sections
  4. Fill in your content using the components

Site Anatomy

Every site follows the same three-panel layout. This page is a live example of that layout.

Multi-page architecture

Each site uses separate .html files linked via the topbar. The sidebar handles within-page navigation. All pages share the same styles.css stylesheet.

How to Use

Starting a New Site

# 1. Create a docs/ directory in your project
mkdir docs

# 2. Copy required files
cp prodocstore/docs/index.html your-project/docs/index.html
cp prodocstore/docs/logo.svg your-project/docs/logo.svg
cp prodocstore/templates/robots.txt your-project/docs/robots.txt
cp prodocstore/templates/_headers your-project/docs/_headers

# 3. Update the title and badge
#    Change <title> and the .badge text

# 4. Update sidebar nav groups
#    Replace with your own sections

# 5. Set up deploy workflow
mkdir -p .github/workflows
cp prodocstore/templates/workflows/docs-deploy.yml .github/workflows/
#    Edit docs-deploy.yml: change project-name to your repo name

# 6. Push to main - deploys automatically
#    Access protection is created on first deploy

Adding to an Existing Site

Refer to the Components page for all available building blocks. Copy the HTML structure for any component you need.

docs/ must be self-contained

Never place publishing-related files outside docs/. No build scripts, generators, assets, or stylesheets in the repo root. The repo root is for project code. Generators (changelog, sitemap, page-meta) live in ProDocStore and run centrally via the reusable workflow - do not copy them into individual repos. The only file outside docs/ is .github/workflows/docs-deploy.yml, which GitHub requires in its standard location.
