ProDocStore Extension

Browser extension that lets you edit docs sites by chatting with an AI agent, directly from the rendered page. No "open the repo, find the file, clone, edit, PR" detour - prompt, review, commit.

Provider-agnostic

The extension is a chat surface. The actual edits are made by a pluggable backend:

  • Claude - Anthropic API + Octokit directly. No extra infra.
  • OpenAI - OpenAI API directly. Same multi-turn tool-loop as the Claude adapter (list pages, read files, propose edits).
  • GitHub Coding Agent (future) - GraphQL assignment to Copilot Coding Agent / Agent HQ. Requires a user PAT with a Copilot subscription.
  • MCP (future) - any Model Context Protocol server.

Source lives at extension/ in the ProDocStore repo.

Content script (content.ts)
  - Reads current page (URL, title, HTML, visible text)

URL to source file resolution

Each docs site declares its backing GitHub repo via a single meta tag in the rendered HTML: <meta name="docs-repo" content="owner/name">. The extension reads this tag and uses it to fetch source files, propose edits, and open PRs. Sites built with ProDocStore templates emit the tag automatically at deploy time. Cloudflare Pages clean URLs (no .html suffix) and trailing slashes are resolved automatically.

Structural (bulk) edits

A dedicated mode for changes spanning every page - nav updates, footer changes, meta tags. The agent produces a single structural diff; the extension applies it to every HTML file under docs/ before opening one PR. This solves the "menu updates" pain point we kept hitting manually.

Install

The extension is not yet on the Chrome Web Store. Install the packaged release zip, or build from source for development.

Prerequisites

  • Chrome 120+ (or any Chromium-based browser with Manifest V3 and chrome.sidePanel: Edge, Brave, Arc).
  • Node 20+ and npm 10+ to run the build.
  • A local clone of ProDocStore-online/platform.

Packaged release

Download the latest ProDocStore extension release, unzip prodocstore-v*.zip, then load the unzipped folder in Chrome.

Build from source

git clone git@github.com:ProDocStore-online/platform.git
cd prodocstore/extension
npm install
npm run build

The build writes a loadable extension into extension/dist/. This is what Chrome reads - not the src/ TypeScript.

Load into Chrome

  1. Open chrome://extensions.
  2. Toggle Developer mode (top-right).
  3. Click Load unpacked.
  4. Pick the extension/dist/ directory (not extension/).
  5. Pin the extension to your toolbar so the icon is always visible.

The extension now appears as ProDocStore.

First run

  1. Open any ProDocStore-enabled docs site.
  2. Click the ProDocStore toolbar icon. Chrome opens a side panel.
  3. The banner under the header shows the repo + source path resolved from the current URL, e.g. your-org/your-repo - docs/index.html.
  4. Pick a backend in the top-right dropdown.

Update after new builds

dist/ ships as a regular folder - Chrome reads it live. After npm run build finishes, click the Reload arrow on the ProDocStore card in chrome://extensions. No need to remove and re-add.

Uninstall

chrome://extensions → Remove on the ProDocStore card. Settings stored in chrome.storage.sync are cleared automatically.

Backends

Claude

  1. Open the extension Options (right-click the toolbar icon → "Options").
  2. Paste an Anthropic API key.
  3. Sign in with GitHub (device flow) or paste a PAT with repo scope. Auth is shared with the OpenAI backend.
  4. Pick a model (e.g. claude-sonnet-4-6).

OpenAI

  1. Extension options → OpenAI section.
  2. Paste an OpenAI API key (the same GitHub auth from the Claude block applies).
  3. Pick a model (e.g. gpt-5.4).

Authentication

The extension itself has no login. It is fully client-side. Auth is only needed for the external services it calls, and every credential is supplied by the user.

Viewing docs sitesCloudflare Access SSO sessionAlready required to open *.pages.dev

Storage

Credentials are written to chrome.storage.sync by the extension's own service worker. Chrome encrypts this at rest and scopes it to your browser profile. If you're signed into Chrome with a Google account, the settings sync across your Chrome installs.

Privacy

The extension sends network traffic only to:

  • The APIs you have configured (Anthropic, OpenAI, GitHub).
  • The current tab's page (read-only, to grab HTML and visible text).

No telemetry, no phone-home, no shared server.

Development

cd extension
npm install
npm run build       # bundles src/ into dist/
npm test            # 29 tests, under 3 seconds
npm run dev         # esbuild in --watch mode
npm run typecheck   # tsc --noEmit

Each save rebuilds dist/. Click Reload once after dist/ is current; further TS edits rebuild automatically, but you still need the Reload click for the service worker and content script to refresh.

Tests

  • tests/resolver.test.mjs - URL → source file mapping, including clean URLs, fragments, and the project-to-repo quirks (12 tests).
  • tests/adapters.test.mjs - registry correctness, validation, AdapterError.cause wiring (9 tests).
  • tests/settings.test.mjs - deep merge so partial updates don't wipe sibling fields (8 tests).

CI runs them on every PR via .github/workflows/test.yml alongside the Python tests for the generators.

Status

Both adapters are wired to real APIs. Claude and OpenAI both call live APIs and drive a multi-turn tool loop (list pages, read repo files, propose edits, open PR or push direct). Per-repo chat history, scroll memory, dictation, theme/font controls, and a diff-preview Apply/Cancel flow are all in place. The next milestone is Chrome Web Store publication and a managed gateway so non-technical users don't need to bring their own API keys.
